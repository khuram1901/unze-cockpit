How the layout reflows

Desktop → Tablet → Mobile
The exact same components appear on both breakpoints — no separate mobile design language. Only the container widths, grid columns, and a handful of density rules change. Every token comes from assets/tokens.css.
01
Left sidebar → bottom tab bar
On desktop, a 220px persistent nav lives on the left. On mobile, the top 5 destinations become a bottom tab bar with icons + labels. All other nav items move behind a "More" sheet from Teams.
02
Hero + 3 tiles → hero + 2×2 grid
Desktop shows the closing-balance hero (2× width, dark card) beside three status tiles. Mobile promotes the hero to a full-width dark card, and the three tiles become a 2×2 grid — the 4th tile is "Net today" so no info is lost.
03
Chart + summary → stacked
The cash-flow chart uses 60% width paired with a quarterly summary on desktop. On mobile, only the chart is shown; the quarterly summary moves to a dedicated Cash tab. Legends collapse from 3 items to 2 (inflow/outflow only).
04
Entity cards → stacked cards
Two entity cards side-by-side become one column. Inside each card, the 3-metric row becomes a 2×2 grid — Cash / Payables on row 1, Receivables / Aging on row 2. Same numbers, same hierarchy, no horizontal scroll.
05
Wide table → card-per-row
The receivables table is one of the trickiest. On desktop it's a 6-column table with summary bar. On mobile, it becomes: 2×2 summary card + a list of invoice rows (status dot / company + due date / amount + aging). No horizontal scroll, ever.
06
Department scorecard → compact list
Desktop shows all 9 departments with segmented task-load bars, owners, and counts. Mobile shows only status dot + name + one-line context + count — sorted by attention needed (danger → warning → success). Full data lives on the Departments tab.
·
Type scale scales down
Hero number 52px → 44px. Section header 20px → 17px. Body 14px → 13.5px. Kicker labels stay 10–11px on both to preserve their spacing rhythm.
·
Card radius scales down
Cards on desktop use 14px radius (20px for hero). On mobile everything nudges up to 16–20px so the softer edges feel more phone-native — matches the Nest / Apple Fitness feel.
·
Density rule
Desktop inner card padding: 22–24px. Mobile inner card padding: 16–20px. Between-card gap: 16px desktop, 12px mobile. Page padding: 40px desktop, 20px mobile.

Reusable system — locked

Open Design System →
Every subsequent page you share will use the same set of components introduced here — Card, Metric card, Delta chip, Status dot, Segmented bar, Progress bar, Data table, Section header, Tab bar, Sidebar. Colors, type, radius, spacing all come from tokens.css. No page needs to introduce new primitives; if it does, we add it to the system.
