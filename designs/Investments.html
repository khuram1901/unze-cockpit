How the layout reflows

Desktop → Mobile
The densest table so far. Instead of horizontal-scrolling the 11-column table on mobile, it becomes a card list with the essential columns only. Cards marked NEW are additions to the design system.
01 · NEW
Ticker chip
Small letter mark (24×24 on desktop, 28×28 on mobile) + mono ticker string. Used everywhere a stock appears — table cells, performer tiles, mobile list, alert chips. Consistent visual anchor.
02 · NEW
Return chip (small)
A tighter variant of the delta chip — 2×8 padding, 6px radius, tabular-nums. Used inline in tables where the standard pill-shaped chip would break the row rhythm. Same success/danger tint tokens.
03 · NEW
Diverging horizontal bars
A zero-axis in the middle; positive bars grow right in success, negative bars grow left in danger. Label on left, value on right — the bar is the middle 60% of the row. Row height 26px on desktop, 22px on mobile.
04 · NEW
Performer tile
Compact card: ticker chip + big return number + underlying value in a muted line. 5-across on desktop, becomes a horizontal-scroll strip of 4 wide cards on mobile — much better than trying to fit them 2-across.
05 · NEW
Ticker selector strip
Horizontal-scroll row of mono ticker chips above a chart. Active chip is dark (ink-900). Reused wherever the user needs to pick "which one am I viewing?" — could apply to bank facilities, entities, cost centers.
06 · NEW
Sticky total row
Tinted (card-alt) background, 2px top border, bold weight, display-font total number, matching return chip. Sits at the bottom of any long table. On mobile the table becomes a card list, and the totals live in the hero card at the top instead.
07
11-col table → 3-col card list
On desktop, holdings are shown across all 11 columns — power users like the density. On mobile, each holding becomes a 3-column row: ticker mark / ticker+company+qty / value+return. That's the essential story of one holding.
08
Actions column → row tap
Desktop has an inline actions cell (view / edit / delete icon buttons). Mobile drops the column entirely — tapping a row opens a detail sheet with the same actions as buttons. FAB handles "Add Holding" instead of a top-right button.
09
Two-chart row → stacked
Price history + diverging chart sit side-by-side on desktop. On mobile they stack, with the ticker selector strip becoming a filter above the (currently smaller) price chart. The diverging chart trims to top 7 rows on mobile.
