How the layout reflows

Desktop → Mobile
The biggest reflow challenge in the app so far. Kanban doesn't work on a 390px screen — the mobile experience becomes a filterable list. Cards marked NEW are additions to the design system.
01 · NEW
Stage board (kanban)
8 columns on card-alt background, each column has: numbered name, live count chip, budget line, and a stack of bill cards. Cards are draggable between columns. Horizontal scroll if the viewport is narrower than ~1440px.
02 · NEW
Bill card
Compact card inside each stage column: customer (mono, bold) + status badges, big amount, issue date, "N d in stage" with danger tint if stuck. Bottom row has Back / Next (primary) / Collected buttons. Extends to any workflow item.
03 · NEW
Bill status badge
Tiny 9px pills that stack in a bill's top-right: Retention (accent), Sales Tax (warning), STUCK (danger). Distinct from status dots — badges name a specific quality, dots signal general state. Reusable in any card.
04 · NEW
Pipeline flow bar
Horizontal segmented bar showing distribution across stages. Each segment is sized proportionally to its count and colored by health (success / warning / danger / neutral). Legend below maps swatches to stage names and counts.
05 · NEW
Velocity strip
A row of 8 metric cells, each showing average days in a stage against its budget. Over-budget cells turn danger red, at-zero cells go muted grey. Great for spotting where the process actually gets stuck.
06 · NEW
Progress tile (accent rule)
A KPI variant with a 3px colored left rule — used for the Total / On-time / Due-soon / Stuck row. Same pattern as the grouped input card from page 05, but at KPI density. Reusable for any 4-way status summary.
07
Board → filterable list
On mobile the 8-column board becomes a horizontal-scrolling stage-filter chip row (All / stage names with counts) above a single-column bill list. Tapping a chip filters, tapping a card opens a detail sheet.
08
Bill card mobile shape
Same anatomy as the desktop bill card, but wider — customer/badges at top, "Stage · N d in stage" line with a status dot, big amount, one-line deduction/notes, and the three action buttons at the bottom.
09
Everything else stacks
Pipeline bar keeps its shape but slims to 24px; legend becomes a vertical list. Velocity strip collapses (accessible via detail page). Aging 4-wide becomes 2×2. KPI row becomes hero + inline foot stats (like page 01/04's mobile hero).
