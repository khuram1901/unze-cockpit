Reflow & new components

Desktop → Mobile
01 · NEW
6-KPI compact row
A tighter KPI variant (16×18 padding, 26px number) so 6 metrics fit on one desktop row. Reusable for ops-heavy pages where 4 KPIs isn't enough.
02 · NEW
Plant comparison bar
Horizontal bar with layered elements: light target bar (background), colored actual bar (foreground), and empty-state "No data" overlay when no reading exists. Great for multi-plant/multi-entity comparisons against a benchmark.
03 · NEW
Breakage-rate row
Ultra-thin bar with a vertical limit marker at 1.5% (danger red line). Below limit = healthy. Reusable for any rate that must stay under a threshold — defect rate, downtime %, churn %, etc.
04 · NEW
KPI comparison table
7-col table with muted zeros, "—" for no-target rows, and clean RED / No target status pills. Sortable by any KPI column.
05 · NEW
Target-vs-actual row
3-col: plant + subtype (Production / Dispatch), horizontal bar showing actual as % of target, right-aligned "N / target" with a big percentage number below (red if under 60%).
06
6-KPI row → hero + 2x2
Mobile promotes "Good stock" to a full-width dark hero. The remaining 4 today-KPIs become a 2×2 grid; broken stock and machine issues collapse into the hero foot line.
07
Wide table → compact list
The 7-column KPI table becomes a 2-line-per-plant list: name + inline "N / target · Wk N / N" summary + status pill. Full table lives behind "Table →" link.
08
Tabstrip stays
Production / Dispatch / Breakage / My tasks tabs render on both breakpoints — same styling. Chart/data below swaps per tab.
