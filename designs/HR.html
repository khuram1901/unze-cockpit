Reflow & new components

Desktop → Mobile
01 · NEW
Tinted KPI (top rule)
Standard KPI card with a 3px accent-colored rule at the top inset from the sides. Warm-amber = at-risk (open), green = filled, red = stale, blue = summary. Reused anywhere a row of KPIs needs subtle inline categorization.
02 · NEW
Two-form split
Two form cards side by side on desktop (Add position + Create task). Each with a tinted head (icon + title + meta) and a footer with hint + Discard/Save. Reusable for any page that offers multiple quick-action forms.
03 · NEW
Priority-in-select
Priority dropdown shows a colored status dot + label ("● Medium"). Selection preview is inline so the user sees the color before opening the menu.
04 · NEW
Suggested-from-source card
Accent-ruled card surfacing pre-computed suggestions from other pages (e.g. positions mentioned in meeting minutes). Reduces dead-end empty states into gentle onboarding.
05
Two-form → stacked + quick-action buttons
Desktop shows both forms open at once. Mobile shows the "Add position" form inline and offers two big buttons up top (Position / Task) that toggle a full-screen form. Same fields, less pressure.
06
Reuses everything else
Sidebar, breadcrumb, section header, KPI card, empty inline state, segtabs — all from the shared shell. Only new visual atoms are the KPI top-rule variant and the priority-in-select pattern.
