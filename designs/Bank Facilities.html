How the layout reflows

Desktop → Mobile
Reuses everything from page 01. Below, each card marked NEW is a component being added to the shared system.
01
4 KPIs → 2×2 grid
Same as page 01: a row of 4 metric cards on desktop becomes a 2×2 grid on mobile. Label shortens ("Active Guarantees" → "Active"), but the number and status dot stay identical.
02 · NEW
Facility utilisation card
A composite card: bank mark + name, big utilisation % on the right, thin bar underneath, and Limit / Used / Available in a horizontal amounts row. Sub-facilities appear as rows below a subtle divider. Reusable for any bank relationship.
03 · NEW
Breadcrumb
Small crumb (Home › Finance › Bank Facilities) above every deep page on desktop. On mobile it collapses to a single "← Home" back link — one level up, which is all you can act on with a thumb.
04 · NEW
Segmented tabs
Pill-shaped tabs for exclusive state (Active / Expired / Pending / All). Dark active pill mirrors the primary button styling — same ink-900. On mobile, tabs collapse into a horizontal-scrolling chip filter with active chip highlighted.
05 · NEW
Filter toolbar
Search input + pill selectors + "Filter" button, right-aligned. On mobile, the search moves into the top nav (magnifying glass icon), and the pills become the chip strip. Same information, less chrome.
06 · NEW
Empty state
Not the current "No guarantees found. Try clearing filters." line — instead a proper empty card: soft-tinted icon, headline, one-sentence body, primary + secondary action. Used wherever a list is empty (guarantees, receivables, alerts).
07
Primary actions
Desktop: "New Facility" (secondary) + "New Guarantee" (primary, dark) sit in the top-right of the page header. Mobile: only the primary action; it becomes both a floating action button (FAB) and the empty-state CTA. Secondary lives in the "…" more menu.
08
Sub-facility list
Desktop: 6-column row inside the facility card (status dot / name & meta / limit / used / available / status chip). Mobile: single line with name on left, "limit · used · avail" mono string on right — tap opens a detail sheet.
09
Density delta
Sub-facility rows: 18px vertical padding on desktop, 14px on mobile. Facility percent number: 44px on desktop, 26px on mobile. Same visual weight relative to its container in each context.
