Reflow & new components

Desktop → Mobile
01 · NEW
Member row
Gradient avatar circle (34px), name + monospace email stacked, department, location, role badge. Reusable anywhere users appear as a list.
02 · NEW
Role badge
Small colored pill. Admin=dark, Manager=accent, HR=warning, Exec Asst=purple, Member=neutral. Different from status pills — role is stable, status changes.
03 · NEW
Access matrix
Dark-header card with a horizontally-scrollable table: rows = roles, columns = modules grouped by section. Cells show green tick (full), amber R (read-only), or dash (none). Row headers pinned left.
04 · NEW
Ownership grid
Dark-header card with rows per department: dept name + HOD + Deputy + Backup selectors. Empty slots show "Not set" in ink-400. Escalations flow through this hierarchy.
05
Segmented tabs (page-level)
People / Access / Ownership / Pending — a soft pill group at the top of the page. Different content per tab. Reuses the segtabs style from Bank Facilities.
06
Mobile: list + chip tabs
Members table becomes a compact list (avatar + name + dept + role badge). Access matrix and ownership grid live behind tabs — they're too wide to reflow, so a dedicated screen with horizontal scroll for the matrix is the honest answer.
